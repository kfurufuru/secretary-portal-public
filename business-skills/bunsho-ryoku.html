文章力の5要素

書くことは考えること。曖昧な文章は曖昧な思考の証拠。結論先行・構造化・具体化・読者分析・編集力の5技術でビジネス文書を再現可能な設計に変える。

①結論先行は土台。読み手が最初の5秒で判断できる文書を作る。「読み終わらないと結論がわからない」構造を根絶する出発点。
②構造化（ピラミッド原則）で主張→根拠→事実の階層を作る。So What?/Why So?の検証が通れば、文章の論理に穴がない。
③具体化・数値化で情報を判断材料に変える。「早めに・コストが大きい・適切に」という曖昧語は情報ゼロ。5W1Hと数値に置き換える。
④読者分析で「誰に・何を届けるか」を起点にする。同じ事実でも相手が役員か部下かで文書の構成・語彙・強調点は全て変わる。
⑤編集（削る技術）で初稿を最終稿に変換する。30〜40%削れる文章は構造と具体化が機能した証拠。削ることへの抵抗感を捨てる。

文書構造の3大原則

すべてのビジネス文書に共通する再現可能な設計原則

PREP法

Point（結論）→ Reason（理由）→ Example（事例）→ Point（再強調）。営業提案・説得文に最適。「何を勧めるか→なぜか→具体例→もう一度結論」の順で読み手の納得を引き出す。結論に到達するまでの階段がシンプルで、迷わない。
✓ メール・プレゼン・営業資料 ✓ 説得力が必要な文書

ピラミッド原則

Main Point（頂点）→ Supporting Points（3つの根拠）→ Details（各根拠の事実）。複雑な情報・重い判断を扱う文書に最適。各根拠は「So What?（だから何）」と「Why So?（なぜそう言えるか）」で検証できる構造を持つ。主張の説得力が高い。
✓ 稟議書・報告書・提案書 ✓ 意思決定が必要な文書

1文1意図

1文に1つの情報だけを乗せる。「〜であり、かつ〜であることから〜のため、〜です」という複文を解体して「〜です。〜です。〜のため、〜します」と分割。文ごとに行動・判断が明確になり、読み手の負荷が下がる。特に指示書・手順書で有効。
✓ 指示書・手順書・マニュアル ✓ 読みやすさ優先
3大原則の選択基準
読み手が忙しく、すぐに判断したい → PREP法または1文1意図。結論が先に見える。
複雑な情報・複数の根拠がある → ピラミッド原則。階層構造で情報量を整理できる。
読み手の知識レベルが低い・専門外 → 1文1意図。シンプルさが理解を助ける。
意思決定者・経営層向け → ピラミッド原則。根拠の論理性が信頼を生む。
説得・提案が目的 → PREP法。最後に再度結論を打つことで記憶に残りやすい。
⚠ 原則を無視した場合の実損
PREP法を無視：営業提案で理由から始まると、読み手が「この話の目的は何か」を推測しながら読む。途中で結論が出ない文書は脱落リスク70%上昇（HubSpot 2023年調査）。

ピラミッド原則を無視：根拠なく「○○すべき」と書くと「証拠は？」と言われる。稟議書で根拠が不足していると差し戻し確実。意思決定の時間が3倍になる（McKinsey研究）。

1文1意図を無視：複文が多い指示書は、読み手が「この文は何をやれと言っているのか」を何度も読み返す。実施ミスや解釈違いが発生。実装の時間が40%増加（工業規格IEC61508より）。

文書タイプ別設計ガイド

4大文書タイプの構成パターン・言葉づかい・チェックポイント

報告書（Report）

構成パターン
背景→課題→調査内容→結果→結論

言葉づかい
「報告いたします」「確認されました」「推奨します」

チェック
✓ 意見・推測を混ぜていない（事実のみ）
✓ 結論が最後の1行に要約されている
✓ 数値・根拠が全て明記されている
✓ 背景は簡潔（背景：2行、内容：8行の比率）

提案書（Proposal）

構成パターン
読み手の課題→提案内容→効果・ROI→条件・期間→決定期限

言葉づかい
「ご提案します」「メリット」「デメリット」「推奨する理由」

チェック
✓ 読み手の利益（What's in it for you）が冒頭
✓ ROI・効果が数値で示されている
✓ 選択肢がある場合は比較表を入れる
✓ 決定期限が明確（〇月〇日まで）

指示書（Directive）

構成パターン
背景（なぜ）→要求（何を）→手順（どうやって）→期限・連絡先

言葉づかい
命令形（～してください）・箇条書き・1文1動作

チェック
✓ 「なぜこれをやるか」が冒頭1〜2行
✓ 各ステップが1動作（複合動作を分割）
✓ 判断が必要な箇所は「〇〇の場合は△△」と場合分け
✓ 不明な時の連絡先が明示

メール（Email）

構成パターン
件名：結論・依頼事項 | 本文：1行結論→背景→具体的内容→期限

言葉づかい
敬語は最小限。改行多用。段落は3行以下。

チェック
✓ 件名で「何が来たのか」が判断できる
✓ 本文1行目に「結論・依頼」がある
✓ 全文が見出しなしで読める（スクロール不要が目安）
✓ 相手の返信期限が明示
✗ 全文書タイプで禁止
曖昧語：「早めに・適切に・十分に・可能な限り・検討する」は使わない。
複文：1文が25字を超えない。超える場合は分割。
敬語過多：「させていただき」「いただきたく」は最小限。読みにくさが上がる。
受け身：「確認されました」より「確認しました」。主語を明確に。
クッション言葉：「恐れ入りますが」「恐縮ですが」を削除。簡潔さが優先。
