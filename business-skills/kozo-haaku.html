5要素の統合：構造を見る力はどう積み重なるか

①反応停止（観察の確保）
→
②幹の発見（支点特定）
→
③立体化（関係把握）
→
④問いの設計（層の分解）
→
⑤往復（再現性化）

①は土台。反射で動いている間は、構造が目に入らない。一拍置いて「何が起きているのか」を見る習慣がなければ、②〜⑤は全部空転する。
②は効率の源泉。力をかける場所を間違えると、頑張っても前進しない。20%の幹を見つけることで、80%の結果に近道できる。
③は深度。点で見るか立体で見るかは、問題解決の解像度を決定的に変える。再発防止は③なしには不可能。
④は精度。問いの層を1段上げるだけで、打ち手の有効性が跳ね上がる。「現象の問い」で終わると、永遠に同じ問題が繰り返される。
⑤は再現性。①〜④を繰り返すだけでは経験が蓄積されない。抽象化と再具体化の往復が、経験を知恵に変える唯一の回路。

最初に鍛えるべきは①。反応を止めなければ、構造は永遠に見えない。

Deep Dive A

構造把握の認知科学：チャンキングと作業記憶

「情報が多すぎて整理できない」の正体は、作業記憶の容量限界にある。George Miller（1956）が発見した「7±2則」とチャンキング理論が、なぜ構造化が認知負荷を下げるかを説明する。構造化はスキルではなく、脳の制約への対処設計だ。

🧠
7±2則（Miller's Law）
出典：G.A.Miller「The Magical Number Seven」(Psychological Review, 1956)
人間の作業記憶（ワーキングメモリ）が同時に保持できる情報の「塊（チャンク）」は5〜9個。電話番号が7〜8桁に設計されているのはこの制約による。会議で10個以上の情報を同時提示すると、参加者の理解保持率は急落する。
10項目のリストを提示した場合、3日後の再現率は平均17%（3〜4項目の構造化グループに分けると68%）（Bower et al., 1969）
🔗
チャンキング理論
出典：Chase & Simon「Perception in Chess」(Cognitive Psychology, 1973)
チャンキングとは、複数の小さな情報を「意味のある塊」として1チャンクに圧縮する認知操作。チェスの達人はボードの配置を個々のコマではなく「攻撃パターン」という塊で認識する。構造化とは意識的なチャンキング設計だ。
エキスパートは同じ情報を5〜7チャンクで認識。初学者は20〜30の断片として処理 → エキスパートの作業記憶使用量は初学者の1/4
⚡
認知負荷理論（CLT）
出典：Sweller「Cognitive Load During Problem Solving」(Cognitive Science, 1988)
認知負荷は「内在的負荷（課題の難易度）」「外在的負荷（情報の提示方法）」「生成的負荷（学習処理）」の3種に分類される。構造化は外在的負荷を削減し、本質課題への認知リソースを最大化する設計手法だ。
非構造化資料での会議は構造化資料比で意思決定エラー率が2.4倍（Clark & Sweller, 2006）。外在的負荷が高い状態では重要判断を避ける傾向が38%増加。
📐
構造化の作業記憶解放効果
実務への適用：構造把握力の本質的根拠
情報を「カテゴリ×優先度×因果」の3軸で構造化すると、同じ情報量でも作業記憶の占有チャンク数は平均で1/3以下に圧縮される。解放された認知リソースが「判断の質・創造性・リスク感知」に回る。
構造化された会議資料を使用したチームは、意思決定速度40%向上・決定後の変更率51%減少（McKinsey Org Design, 2023）

Deep Dive B

複雑情報を構造化する5ステップ

「なんとなく複雑」の状態から「支点が見えた」状態へ移行するための標準プロセス。頭の中だけで行うと作業記憶の限界に阻まれる。各ステップで外部化ツールを使うことが必須条件だ。

1
全体収集 — まず「全部出す」
頭の中にある情報・懸念・関係者・制約をすべて外部化する。判断・評価・分類は一切しない。「書いていないものは存在しない」前提で、とにかく発散する。この段階でフィルタリングすると重要な要素を見落とす。時間目安：5〜10分。
ツール: ポストイット / マインドマップ / ブレインダンプメモ
2
要素分解 — 「これは何の話か」を問う
収集した情報を「人 / 物 / 金 / 時間 / 情報」などの基本軸、または「原因 / 症状 / 影響 / 対策」の因果軸でカテゴリに分ける。1つの要素が複数カテゴリに属する場合は主眼で決定し、二重計上を避ける（MECE原則）。
ツール: MECE分解 / Issue Tree / KJ法
3
関係性マッピング — 「何が何を動かしているか」を図示
要素間の「因果・依存・時系列・重複」関係を矢印で表現する。この段階で初めて「ボトルネック」「レバレッジポイント」「循環ループ」が見えてくる。線形リストでは発見できない構造的洞察がここで得られる。フィードバックループの存在に特に注意する。
ツール: 因果ループ図 / システム思考マップ / 矢印付きA3整理
4
階層整理 — 「粒度を揃える」
マップから「戦略レベル / 戦術レベル / オペレーションレベル」の階層を分離し、各階層内の情報が同じ抽象度であることを確認する。異なる粒度の情報が同列に並んでいる状態が「混乱した議論」の構造的原因だ。ピラミッド原則の縦軸（抽象度）を揃える作業。
ツール: ピラミッド構造図 / ロジックツリー / 抽象度スケール
5
要約 — 「一言で言うと何か」を結論化
ステップ4までで整理した構造を「この問題の本質は○○だ」という1文に圧縮する。1文にできない場合は、まだ構造が見えていないサインだ。この1文が会議の冒頭・報告書の冒頭・稟議の要旨になる。抽象化思考の完成形がここにある。
ツール: BLUF文の作成 / エレベーターピッチ30秒 / 新聞の見出し作成法
